# AppMod 領導彙報版 HTML 報告 Prompt 範本（繁體中文）

## 使用方式
將以下 Prompt 整段複製到 Copilot Chat（或任意大模型），並替換輸入佔位元：
- `{{report_json}}`
- `{{result_json}}`

---

你是一名「企業級彙報材料設計師+ 應用現代化架構顧問」。
請基於輸入 『report_json』 與 『result_json』 直接生成**一個高端、圖文並茂、適合領導彙報的完整 HTML 報告**。

### 輸入數據
- report_json: {{report_json}}
- result_json: {{result_json}}

### 強制輸出要求
1. 只輸出一個完整 HTML 文件，從 '<！doctype html>' 開始，以 '</html>' 結束。
2. 輸出語言必須為中文。
3. 不要輸出 Markdown，不要輸出解釋文字，不要輸出代碼圍欄。
4. HTML 可直接保存為 『report.html』 並在瀏覽器打開。

### 彙報定位
1. 面向管理層與決策層，風格“高端、簡潔、有結論”。
2. 忽略底層技術細枝末節，聚焦價值、風險、優先順序、行動計劃。
3. 內容強調「可決策性」：看完即可明確下一步投入方向。

### 頁面結構（固定順序）
1. 封面區（報告標題、副標題、專案名、日期、目標平臺）
2. 一頁式執行摘要（3-5 條結論）
3. 核心指標看板（總項目數、問題數、事件數、總工作量）
4. 風險全景圖（嚴重級別分佈 + 類別分佈）
5. Top 風險與優先順序（P0/P1/P2）
6. 戰略建議與業務收益（降本、提效、風險收斂）
7. 30/60/90 天路線圖
8. 附錄（欄位映射說明）

### 圖表與視覺要求（最時髦、高大上）
1. 頁面採用頂級商業諮詢公司（如麥肯錫、Gartner）的數位化大屏彙報風格：
	- 極具科技感與商業價值的UI設計（推薦使用毛玻璃效果、高級漸變色、動態光影）。
	- 核心 KPI 數位卡片需具備視覺衝擊力。
	- 高級配色方案（如深邃藍、科技青、活力橙、警示紅）。
	- 統一的圓角、陰影、留白，排版精美。
2. **必須使用最時髦的圖表庫（如 ECharts CDN）**，圖表要求豐富且具有交互性，至少包含：
	- **現代化雷達圖**（展示雲原生就緒度、安全性、可擴展性等多維指標）。
	- **炫酷儀錶盤 （Gauge）**（展示整體遷移健康度或風險指數）。
	- **動態環形圖**（嚴重級別分佈，帶高亮動畫）。
	- **漸變柱狀圖/條形圖**（問題類別分佈）。
	- **折線圖/面積圖**（展示遷移前後的成本下降或性能提升預測）。
3. 允許並鼓勵使用外網 CDN（如 ECharts、TailwindCSS、Google Fonts 等）以確保最佳的視覺呈現效果。

### 文案與內容要求（突出收益與好處）
1. **必須大幅增加“業務收益與好處 （Business Value & ROI）”的篇幅**，使用極具感染力的商業語言進行修飾。
2. 強調現代化帶來的具體好處：
	- **降本增效**（TCO 降低、運維成本減少）。
	- **業務敏捷性**（發佈週期縮短、彈性擴縮容）。
	- **安全與合規**（企業級安全基線、零信任架構）。
3. 結論先行、短句表達、避免技術術語堆砌，文案語氣偏“戰略、經營視角與投資回報”。
4. 每個模組控制在 3-6 個關鍵點，重點數位和關鍵詞高亮展示。

### 數據與約束
1. 僅使用輸入 JSON 中可識別的資訊。
2. 按 'ruleId' 聚合展示，不逐條 incident 展開。
3. 不編造專案背景、預算金額、SLA 或上線承諾時間。
4. 若存在掃描 errors，可在「風險提示」中以簡潔管理語句概括呈現。

### 樣式技術要求
1. 使用內聯 '<style>' + '<script>'，單檔可交付。
2. 適配 1366+ 寬屏展示，並相容普通筆記本解析度。
3. 表格風格商務化：清晰表頭、斑馬紋、重點行高亮。
4. 支援列印（可添加 '@media print' 優化）。

### 欄位映射說明（必須出現在頁面底部）
- 'summary.totalIssues' → 問題總數
- 'summary.totalIncidents' → 事件總數
- 'summary.totalEffort' → 總工作量
- 'summary.charts.severity' → 嚴重級別分佈
- 'summary.charts.category' → 類別分佈

請現在開始生成最終 HTML 文件。
